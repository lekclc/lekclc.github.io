protobuf

Protobuf

介绍

一种数据压缩编码方式,用于通信协议，数据存储等

对于数据结构,根据目标实现将其编译为代码或另一种结构

只需使用 Protobuf 对数据结构进行一次描述，即可利用各种不同语言或从各种不同数据流中对结构化数据读写

Protobuf官方并不支持C语言,需要使用一个第三方库 protobuf-c

https://github.com/protobuf-c/protobuf-c

工具

示例

syntax

指明protobuf的版本，有proto2和proto3两个版本，默认为proto2

package

防止命名空间冲突，简单项目可以省略

message

定义消息结构体，类似C语言中的struct

修饰符

  • optional：可以不提供字段值，字段将被初始化为默认值。（Proto3中不允许显示声明，不加修饰符即optional）
  • repeated：类似vector，表明该字段为动态数组，可重复任意次。
  • required：必须提供字段值。（Proto3不再支持required）

基本类型

  • bool
  • in32
  • float
  • double
  • string

需要关注的有

  • magic：通常为0x28AAEEF9。
  • n_fields：结构体中的字段数量。
  • fields：指向一个储存字段和数据的结构体。

fields是ProtobufCFieldDescriptor类型

ProtobufCFieldDescriptor

  • name：字段名。
  • id：唯一字段编号。
